  • LSA即Local Security Authority（本地安全机构）
  • 对应lsass.exe进程，负责处理本地登录、密码策略、域认证等一切和身份验证相关事物
  • Windows 在你第一次登录成功后，会把你的明文密码（早期系统或特定配置下）、NTLM Hash、以及 Kerberos 票据，缓存 lsass.exe 的内存空间里
  • 需要SYSTEM权限才能访问读取，具体来说是需要SeDebugPrivilege（调试特权）
    • [[用户与用户组]]
    • 本地管理员组的成员是默认拥有 SeDebugPrivilege 的
  • 拿到Hash后就能进行哈希传递攻击（Pass-the-Hash）
  • WIN11默认开启了LSA保护，即将其作为PPL（受保护的轻量级进程）去运行
    • 只有带有微软特定数字签名的其它PPL进程才能读取LSA内存
    • 关闭防火墙PPL还会起作用

NTLM

  • NTLM即NT LAN Manager，是微软早期发明的一套身份验证协议和哈希算法
  • 参考：域渗透——Pass The Hash的实现
  • 在没有域环境的电脑中，或者当域环境内Kerberos认证失败、不支持 IP 直接访问时，系统会降级默认使用 NTLM 认证
  • hash分为LM hash和NT hash，如果密码长度大于15，那么无法生成LM hash。从Windows Vista和Windows Server 2008开始，微软默认禁用LM hash

NTLM认证机制

  • 客户端请求登录
  • 服务器发送质询（一串随机数据）
  • 客户端使用NTLM Hash对质询进行加密并返回
  • 服务器比较哈希

NTLM Hash

  • 只有Windows早期系统可能会在本地数据库（SAM文件）或域控数据库（NTDS.dit）
  • 因此Windows会将密码进行NTLM哈希算法编码得到NTLM Hash
LMHASH与NTHASH
  • LM HASH (LAN Manager Hash): 这是一个非常古老、极其脆弱的加密算法（最多支持 14 个字符，且不区分大小写，还会被切成两半加密）。因为它太容易被暴力破解，从Windows Vista和Windows Server 2008开始，微软默认彻底禁用了 LM Hash 的存储。
  • NT HASH (NTLM Hash): 这是现代 Windows 系统中用来存储密码和进行网络认证的核心凭据。算法更安全（基于 MD4），支持更长和更复杂的密码

哈希传递攻击

  • 众所周知，现在基本都不直接储存明文密码，而是通过各种各样的哈希算法去编码储存，在输入密码验证的最后就是进行哈希比较验证
  • 而有的服务能直接进行哈希认证而不需要输入明文密码，就类似于SSH的公钥认证，而Windows认证中也是这样的一种服务

Procdump

  • 微软自己做的工具，官方介绍：ProcDump 是一个命令行实用工具，其主要用途是监视应用程序的 CPU 峰值，并在出现峰值期间生成故障转储，管理员或开发人员可以使用这些转储来确定出现峰值的原因。 ProcDump 还支持挂起窗口监视（使用与 Windows 和任务管理器使用的窗口挂起相同的定义）、未处理的异常监视，并且可以根据系统性能计数器的值生成转储。 它还可用作可嵌入到其他脚本中的常规进程转储实用工具
  • 可以用来把lsass.exe的内存dump出一个.dmp文件，然后拿去给Mimikatz 或 pypykatz 读取
