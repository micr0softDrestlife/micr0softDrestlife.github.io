  • WebDAV（Web-based Distributed Authoring and Versioning，基于Web的分布式编写和版本控制）是一种基于HTTP的通信协议
  • 它扩展了HTTP，在GET等几个HTTP标准方法以外添加了一些新的方法，使应用程序可对Web Server直接读写，并支持写文件锁定及解锁，还可以支持文件的版本控制
  • WebDAV有利于用户间协同编辑和管理存储在万维网服务器文档
  • Windows使用WebClient服务实现WebDAV，允许Windows应用程序通过使用WebDAV来创建、读取和写入Internet文件服务器上的文件，并使用HTTP进行通信

触发跨协议中继

  • 使用特殊的UNC路径
  • 当对启用WebDAV的UNC路径触发文件操作时，客户端主机将与WebDAV服务器执行如下交互过程：[[NTLM Relay over HTTP (Webdav)]]
  1. 客户端发出一个OPTIONS方法来发现服务器支持的请求方法。
  2. 如果支持PROPFIND方法，则发出PROPFIND请求来发现目录结构。
  3. 如果服务器以401 Unauthorized响应并通过WWW-Authenticate标头请求NTLM身份验证，则WebDAV将继续启动NTLM质询响应身份验证，最终将Net-NTLM Hash提供给服务器
  • 客户端系统上必须安装并启用WebClient服务。尽管这是Workstation版本系统上的默认设置，但并不适用于Server版本系统。在Server版本系统上仍然需要通过附加功能来安装并启用WebDAV组件
  • 但低权限用户仍然可以通过调用服务控制管理器（Services Control Manager，SCM）来启动WebClient服务

探测

  • [[crackmapexec]]

  • 很多运行在 IIS 上的企业应用默认支持 NTLM 认证且没有开启扩展保护（EPA）或签名。

  • 典型代表：AD CS（Active Directory 证书服务） 的 Web 注册页面。这就是著名的 ESC8 漏洞的原理——攻击者把强制触发的 NTLM 流量中继到 AD CS 的 HTTP 服务上，由于 HTTP 不校验签名，攻击者可以直接为高权限账户申请到一张合法的数字证书，进而伪造身份接管域控。

  • Exchange EWS 接口： 以前经常被用来打 Exchange 的特权提升。
