  • 域环境（Domain）是微软为企业网络涉及的一种集中式管理模式。没有域环境的情况下每台电脑的账密等等都存在本地，在域环境下由核心服务器统一管理
  • AD 域（Active Directory，活动目录）：是微软对“域环境”这个概念的具体技术实现，即存储在DC上的核心数据库（存储在域控的 NTDS.dit 文件里）。
    • AD本质上就是一个分布式的数据库，里面存着域中所有的计算机、用户、密码哈希和权限关系
    • 默认路径：C:\Windows\NTDS（大小写皆可）
    • 存放了：ntds.dit、edb.chk（检查点文件，记录了数据库最后一次写入的进度）、edb*.log（事务日志文件）、temp.edb（临时工作文件）等
    • 利用ntds.dit和SYSTEM获取域哈希：[[LSA内存与NTLM]]
  • 在Winodws中域和AD域是等价的
  • 域目录：在展现形式上与文件目录类似，但后者是存在硬盘扇区里用来存储真正的文件数据，而域目录本质是一个分布式关系型数据库（实体文件在DC的ntds.dit中
    • 存储对象与属性，员工账号、加入域1电脑、部门等等都是对象
    • 权限设置、组策略（GPO）等也是对象或对象的属性（比如给账户对象设置一个msDS-AllowedToActOnBehalfOfOtherIdentity属性）
    • [[LDAP与LDAPS]]

域的结构

  • 域管（DA）：是一个域环境中权限最高的人类管理员账号
  • 域控（DC）：是机器，是运行AD数据库的核心机器
    • KDC（密钥分发中心），是DC的服务组件，专门负责Kerberos认证协议的运行，负责核对密码、颁发TGT和ST
      • 默认运行在88端口
  • AD域
  • 组策略
  • 组织单位
  • 域林与域树

域用户

  • [[用户与用户组]]
  • 默认情况下域用户确实可以通过网络去访问域内普通机器的共享，或者进行交互式登录，但这在机器上通常只是标准用户而没有Administrator权限
    • 并且默认情况下普通域用户没有[[RDP]]和[[WinRM]]的权限，要远程控制目标机器的话目标机器的Remote Desktop Users或Administrators组里必须有这个域用户
  • Domain Admins：域管，管理一个域
  • Enterprise Admins：企业管理员，管理整个域林。该组只存在于林根域（即第一个建立的子域）
  • Schema Admins：架构管理员组，和EA一样，也是只存在于林根域的顶级权限组
  • krbtgt：[[凭据与票据]]
域内置组
  • 域内置组提供的是域级别的权限而不是单台机器的本地权限
Domain Admins
  • 域管组
Account Operators
  • Account Operators（账号操作员）：这个组的成员可以在活动目录中创建、修改和删除普通的域用户、域组和计算机账户
    • 可以拿到域控以外所有机器的SYSTEM权限
  • 为了防止越权，微软规定 Account Operators 不能修改那些高级特权组（如 Domain Admins、Enterprise Admins、Administrators、Server Operators 等）及其成员的属性
Backup Operators
  • Backup Operators（备份操作员组）：是为了让员工能备份服务器数据而设立的组
  • 拥有 SeBackupPrivilege 和 SeRestorePrivilege 特权。系统会允许它们绕过所有的文件权限去读取和写入任何文件。常用于获取域控上的 NTDS.dit和 SYSTEM 注册表项
Server Operators
  • Server Operators（服务器操作员组）：是负责维护域服务器的组。
  • 这个组可以登录域控，并且可以启动或停止系统服务或备份和恢复文件。可以通过修改某个以 SYSTEM 运行的服务的执行路径，重启服务后直接获得域控的 SYSTEM 权限。
DnsAdmins
  • DnsAdmins（DNS 管理员组）：是负责管理域内的DNS服务器。在AD环境中，DNS服务通常直接安装在DC上，且以 SYSTEM 权限运行
  • 如果获得一个属于 DnsAdmins 组的普通账户，可以让DNS服务加载指定的恶意DLL文件获得SYSTEM权限 （DNSAdmin EoP）
Print Operators
  • Print Operators（打印机操作员）：可以登录域控并域控上加载设备驱动程序
  • 常结合Print Spooler的漏洞，或者利用加载恶意打印机驱动的方式在域控上执行代码提权

域数据同步

  • 在域中，不同的域控之间，默认每隔15min就会进行一次域数据同步
  • 当一个额外的域控想从其他域控同步数据时，额外域控会向其他域控发起请求，请求同步数据。如果需要同步的数据比较多，则会重复上述过程
  • DCSync就是利用这个原理，[[mimikatz]]

域机器

  • 将一个机器加入域的时候要输入一个域用户的账号密码，这个用户就是将机器加入域的域用户

域扫描

SharpHound (BloodHound 的数据采集器)

  • 作用： 运行后会自动收集域内的用户、计算机、组、ACL（控制列表）、会话（谁登录在哪台机器上）等信息，并打包成一个 .zip 文件
  • 原理是LDAP查询，是通过伪造身份向域控发起查询，因此只需要域控IP
  • 把.zip导入到 BloodHound 的图形化界面中。能直接画出一条从当前普通账户 -> 提权 -> 拿下域管的视觉攻击路线图
  • [[BloodHound]]

PowerView.ps1 (PowerSploit 框架的核心组件)

  • Get-NetDomain (查域基础信息)
  • Get-NetUser (查所有域用户)
  • Get-NetComputer (查所有域内机器)
  • Find-LocalAdminAccess (查当前账户对域内哪些机器拥有本地管理员权限)

ADExplorer (Sysinternals 官方工具)

  • 作用：是微软官方提供的高级 Active Directory 查看器。你可以用它连接到域控，然后点击 “Create Snapshot”，把整个 AD 数据库的快照打包成一个 .dat 文件带走。
  • 优势： 因为是微软官方带有合法签名的工具，几乎没有杀毒软件会拦截它。带回本地后，可以用 ADExplorerSnapshot.py 脚本或自带的离线工具慢慢分析，极其隐蔽
