SMB

  • SMB即Server Message Block，服务器消息块）是 Windows 网络中最核心的协议之一，是微软开发的核心网络文件共享协议
  • 默认端口是445（[[6-端口与默认端口与扫描]]）
  • 主要功能是网络文件共享、打印机共享以及命名管道（Named Pipes）通信
    • 文件传输功能比较重要
  • SMBv1： 极其古老且充满漏洞。永恒之蓝（MS17-010）就是利用了SMBv1的底层设计缺陷
  • SMBv2/v3： 现代系统使用的版本，安全性大幅提升。
  • SMB 签名 (SMB Signing)： 为了防止中间人攻击（如 SMB Relay），Windows 会验证数据包的签名。如果目标机器没开启强制签名（默认域控强制开启，普通域机器不强制），攻击者就可以截获别人的登录请求，并把请求“转发”给目标机器，从而登录目标机器
  • 绝大多数的RPC触发（[[RPC]]）都会默认让目标发起出站的SMB连接
    • 如果在内网的共享文件夹里放置恶意的 .scf、.lnk 文件或修改了 desktop.ini，当管理员浏览该目录时，Windows资源管理器会自动尝试通过SMB加载图标，从而泄露凭证
    • 因为SMB连接过程中会进行进行凭据验证（[[NTLM Relay over HTTP (Webdav)]]）

CIFS（SMB共享）

  • 即Common Internet File System（通用互联网文件系统），是SMBv1的改进
  • CIFS是SMB的曾用名，在Windows的Kerberos认证中，访问SMB共享时，底层申请的SPN名字一律叫 cifs
  • 在后面的改进中该名被废弃，统称为SMB协议，但在Kerberos的底层代码和SPN注册列表仍然存在
  • 域中只要登录了电脑，系统就会自动申请访问DC的cifs服务的ST，这个文件共享服务是始终存在的
    • 但一般只能访问DC上公开的共享目录，除非ST是DC的

命名管道

  • 命名管道是Windows进程间通信的一种方式，就像内存中的一根管子（队列），一个进程往里写数据，另一个进程往外读数据
  • 当命名管道需要跨越网络时，Windows通常会将命名管道包装在SMB协议中传输
  • 很多Windows核心服务就是通过特定的RPC命名管道来对外提供接口的
    • [[RPC]]

默认隐藏共享（C$, ADMIN$, IPC$）： * 只要是 Windows 系统，默认都会开启这几个隐藏的共享目录（名称后面带 $\，普通人在网络邻居里是看不到的）。

  • C$ 代表 C 盘根目录，ADMIN$ 指向 C:\Windows 目录。

  • IPC$（进程间通信）最特殊：它不对应某个具体的文件夹，而是一条“秘密通道”。通过它，可以建立空连接（Null Session），这是很多内网信息收集的基础
    PsExec 依赖于 SMB 协议（TCP 445 端口） 和目标机器的默认隐藏共享（如 ADMIN$ 或 C$）
