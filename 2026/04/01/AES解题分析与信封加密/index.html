AES解题分析与信封加密

  • AES的加密模式：
  • AES 常见长度：
    • 16 字节：AES-128
    • 24 字节：AES-192
    • 32 字节：AES-256

关注点

  1. 模式是什么
  2. key是固定的还是随机的
  3. IV/nonce是固定的还是随机的
  4. 同一个key下IV/nonce会不会重复
  5. 有没有认证机制
  6. 密文是否可被用户提交回服务端
  7. 是否存在chosen-plaintext / known-plaintext条件
  8. 最终能不能转成信息泄露、伪造或提权

事先了解的概念

key

  • 略

IV

  • Initialization Vector，初始化向量
  • 常见于 CBC 这类模式
  • 用于给加密过程提供一个初始变化量，==防止同样的明文和密钥加出一样的密文==
  • 一般不需要保密，但通常要求不可预测或至少唯一

nonce

  • number used once
  • 常见于 CTR、GCM，和IV作用接近，核心都是让同一个密钥的多次加密不会落在同一个加密状态上
  • 一般不需要保密，但在同一个key下必须唯一

tag

  • 认证标签
  • 常见于 GCM/EAX/CCM
  • 用来校验密文、参数和附加数据是否被篡改

AAD

  • Additional Authenticated Data
  • 不加密，但会被认证保护
  • 常见于协议字段、头部、上下文信息

ECB

  • 最不安全
  • 相同明文块会产生相同密文块
  • 会泄露结构特征

利用点

  • 看是否有重复块
  • 看是否能拼接块伪造管理员数据

CBC

  • 需要 IV
  • 只保密，不自带认证
  • 错误的填充校验可能导致 padding oracle

利用点

  • Padding Oracle，看是否返回padding错误
  • 看IV是否和用户输入、cookie、URL 参数有关，IV可控将导致首块明文可控
  • 只加密不验签导致密文可篡改

CTR

  • 即Counter Mode，工作方式不是那AES直接加密明文块，而是：
  1. 把 nonce + counter 当输入
  2. 用AES加密这个输入
  3. 得到一段伪随机输出，叫 keystream
  4. 再用：
    ciphertext = plaintext XOR keystream

  • 所以AES-CTR本质上把AES变成了流密码

  • 在CTR里，norce是生成keystream的起点之一，是每次加密用一次的随机/唯一值

    • 不需要保密，但必须在同一个key下保持唯一，不如可能出问题

  • 能控制加密明文的话就可能能加密全0获得keystream

  • 本质上是流密码

  • 加解密都是异或 keystream

  • 速度快，可随机访问

  • 最大风险：同 key 下 nonce 重复

常见利用点：

  • keystream reuse
  • 已知明文/可控明文恢复 keystream
  • 只加密不认证，密文可按位翻转

GCM

  • 底层加密部分接近CTR思想
  • 但额外加了认证tag
  • 因此不光保密，还能校验数据有没有被改，但前提同样是：key + nonce 不能乱复用
  1. 将明文加密
  2. 生成认证标签tag
  • tag用来校验这份密文是否被篡改，key/norce是否正确
  • 加密后会返回密文和tag，norce一般是库生成的，并且被保存在对应的AES实例中
  • 有时候也会遇到只 decrypt() 不 verify()的情况

  • 它会一次性做两件事：
  1. 加密明文，得到 ciphertext
  2. 计算认证标签，得到 tag

信封加密
  • 可以避免对相同明文产生一样的密文，可以使用多个密钥加密同一个数据，可以结合多种加密算法的优势等等
  • 信封加密就是用一个密钥加密明文数据，然后用另一个密钥加密这个密钥
  1. A和B都持有对称密钥S，传递消息时A临时生成对称密钥R，并==用该密钥加密明文==，然后把密文发给B
  2. 此时B只有密钥S而没有密钥R，无法进行解密。
  3. 利用信封加密思想，A使用S对密钥R进行加密并发给B，这样B就能通过S把R给解密出来了
  • 把R称作内容加密密钥（CEK，Content Encrypting Key）
  • 把S称作密钥加密密钥（KEK，Key Encrypting Key）
  • 当使用KEK加密CEK之后，就不需要再去保存CEK的明文了，也避免了CEK被泄露的风险（除非KEK泄漏了）
  • CEK：开保险箱里的小钥匙
  • KEK：锁小钥匙的总钥匙
